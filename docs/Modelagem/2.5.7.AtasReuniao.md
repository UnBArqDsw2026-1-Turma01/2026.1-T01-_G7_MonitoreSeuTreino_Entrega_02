### Decisões e Deliberações

1. **Repositório da Entrega 2** — criar o repositório da segunda entrega e organizar o workflow de trabalho.
2. **Fluxo de elaboração e revisão** — cada diagrama segue o ciclo: elaboração pelo responsável → revisão por outro membro da equipe.
3. **Prazo de entrega** — todas as atividades devem ser entregues até 24/04 às 08h. Após a apresentação para a professora, será permitido realizar correções até 24/04 às 23h59. Reunião de pré-entrega na quinta-feira.

### Orientações para Contribuição no Repositório

- **Commits atômicos:** cada commit deve ser focado e conter uma mensagem explicando a contribuição realizada.
- **Co-autores:** caso a atividade esteja sendo feita em dupla, adicionar co-authors no commit.
- **Histórico de versão:** atualizar corretamente o histórico de versão de cada página da documentação.
- **Documentação do workflow:** documentar as orientações do workflow no repositório para referência da equipe. O fluxo obrigatório é: criar branch a partir da dev, realizar alterações, atualizar histórico de versão se modificar Markdown, abrir PR para dev (sem review, deve passar no check de histórico), e quando a dev estiver pronta, abrir PR de dev para main (requer 1 aprovação e todos os checks). Não realizar commits diretamente na main.
